Mapa UX/UI + Agile Profesional

Guía paso a paso con tips, herramientas y mejores prácticas

USUARIO / CLIENTE

  • Punto de partida: necesidades reales
  • Dolor del usuario (Pain Points)
  • Objetivos de negocio y KPIs
  • Herramientas: Google Analytics, Hotjar, Typeform

UX

  1. Fundamentos: UX ≠ UI, usabilidad + deseabilidad
  2. Leyes UX: Jakob, Hick, Miller, Fitts Jakob: patrones esperados | Hick: menos opciones | Miller: 7±2 | Fitts: tamaño y distancia
  3. Investigación: Personas, Journey Maps, Benchmarking
  4. Tests de usabilidad: Maze, Lookback, UsabilityHub
  5. Tip: Plantillas de entrevistas y tests para agilizar

UI & Figma

  1. Figma: Auto Layout, Components, Variants
  2. Sistemas de diseño: Atomic Design, Grid 8pt, Spacing consistente
  3. Tipografía: Jerarquía H1-H2-H3, Google Fonts, regla 60-30-10
  4. Iconografía y assets: Iconify, Unsplash, Feather Icons
  5. Accesibilidad: contraste con Stark, colores legibles

Agile

  1. Sprint Planning: priorizar tareas por valor + esfuerzo
  2. Daily: solo bloqueos y avances
  3. Design QA: consistencia visual, interacción y accesibilidad
  4. DoD: cada entrega lista para dev

Práctica / Flujo

  1. Briefing: recopilar info de usuario, objetivos, KPIs
  2. Investigación UX: Personas, Journey Map, Benchmark
  3. Wireframes low-fi: papel o Figma
  4. Prototipo interactivo: Figma, Adobe XD, Maze
  5. Testeo: Maze / Lookback → feedback + métricas
  6. Iterar según hallazgos
  7. UI Final: grids, tipografía, color, iconos, accesibilidad
  8. Handoff: Zeplin / Figma Dev Mode → components + tokens + assets
